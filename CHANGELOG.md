# Changelog

## v1p2 — 2025-05-25

**XPath 定位修正 + Fallback 機制**

### 修正
- **上班/下班按鈕定位錯誤**：舊版 `//div[contains(., '打卡鐘')]` 使用 `.` 比對整棵子樹，導致命中範圍過大，可能誤觸公告區等其他含「上班」文字的元素。改用 `contains(@class,'punch')` 直接鎖定打卡鐘 widget 容器。

### 新增
- **`_find_punch_button()` 方法**：統一管理上班/下班按鈕的定位邏輯，取代分散在各處的單一 XPath 呼叫。
- **4 組候選 XPath Fallback**：依序嘗試，第一個命中即採用，並在日誌中標示候選編號。優先順序根據實測結果排列（class/id 含 `punch` 的容器策略優先）：
  1. `class/id` 含 `punch` 的容器（已確認可用）
  2. `<button>` / `<a>` 語意化元素精確比對
  3. 直接子節點為 `<h3>`/`<h4>` 標題的 div
  4. `<h3>`/`<h4>` 標題後的 `following::` 軸

## v1p1 — 2025-05-25

**上班打卡支援 + 執行日誌 + 啟動前自動測試**

### 新增
- **上班打卡排程**：新增上班/下班雙排程，各自獨立勾選啟用，設定不同時間。
- **執行日誌**：所有操作同步寫入 `punch_log.txt` 並在 UI 日誌區即時顯示（INFO / WARNING / ERROR 三級）。
- **啟動前自動測試**：點擊「儲存並啟動排程」後，先自動執行一次測試模式確認按鈕定位正常，測試通過才啟動正式排程倒數。
- **設定持久化擴充**：`user_config.json` 新增儲存 `checkin_time`、`checkout_time`、`checkin_enabled`、`checkout_enabled` 欄位。
- **智慧倒數 Sleep**：距觸發時間 > 10 秒時以較大步幅 sleep，最後 10 秒改為 0.5 秒細粒度輪詢，減少 CPU 空轉。
- **`threading.Lock` 任務計數**：多執行緒安全地追蹤剩餘任務數，全部完成後才重設 UI 按鈕狀態。

### 修正
- **時間已過自動改隔天**：`_parse_trigger_time()` 偵測到設定時間早於當前時間時，自動順延至隔天同一時間並記錄 WARNING，取代舊版手動加 1 分鐘的行為。

## v1p0 — 初始版本

**僅支援下班打卡的基礎版本**

### 功能
- 透過 Selenium + Chrome 自動登入 NUEIP，於指定時間點擊下班打卡按鈕。
- 測試模式：找到按鈕後以紅框黃底標記，不實際點擊。
- 帳密存入 `user_config.json`（僅 company / user / password / url）。
- 進階 XPath 欄位可自訂各元素定位路徑。
- 倒數等待採固定 0.5 秒輪詢。
