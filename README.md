# 社畜神器

針對 [社畜系統](https://cloud.nueip.com) 設計的自動打卡工具。  
使用 Selenium 自動開啟 Chrome、填入帳密、定時點擊打卡按鈕，支援上下班雙排程與前置測試驗證。

## 環境需求

| 項目 | 版本要求 |
|------|----------|
| Python | 3.8 以上 |
| Google Chrome | 最新穩定版 |
| ChromeDriver | 由 `webdriver-manager` 自動管理，無需手動下載 |

**必要套件：**

```
selenium
webdriver-manager
```

## 安裝步驟

1. 確認已安裝 Python 3.8+：

   ```bash
   python --version
   ```

2. 安裝相依套件：

   ```bash
   pip install selenium webdriver-manager
   ```

3. 執行程式：

   ```bash
   python auto_check_out.py
   ```

## 介面說明

啟動後會出現圖形化視窗（680×960），分為以下區塊：

**第一步：登入資訊（自動記憶）**

| 欄位 | 說明 |
|------|------|
| 公司代碼 | NUEIP 登入頁面的公司代碼 |
| 員工編號 | 你的員工帳號 |
| 登入密碼 | 你的登入密碼 |

> 帳號密碼與排程設定會自動儲存於同目錄的 `user_config.json`，下次啟動自動讀取。

**第二步：排程設定**

| 欄位 | 說明 | 預設值 |
|------|------|--------|
| 登入網址 | NUEIP 登入頁 URL | `https://cloud.nueip.com/login` |
| 上班打卡 | 勾選啟用；填入上班打卡時間 | 未勾選，`09:00` |
| 下班打卡 | 勾選啟用；填入下班打卡時間 | 已勾選，`18:00` |
| 保持測試模式 | 勾選後只標記按鈕位置，不實際點擊 | 預設開啟 |

> 上班與下班可同時勾選，程式會建立兩個獨立排程執行緒同時倒數，互不干擾。

**進階 XPath 定位（選填）**

預設提供 4 組 XPath 候選，依序嘗試，一般不需要修改。如果 NUEIP 改版導致按鈕找不到，再調整這裡。

**執行日誌區**

視窗底部有即時日誌區塊，顯示 INFO / WARNING / ERROR 等級訊息。所有日誌同步寫入同目錄的 `punch_log.txt`，可事後查閱。

## 操作流程

1. 填入公司代碼、員工編號、密碼
2. 勾選要排程的打卡項目（上班 / 下班 / 兩者皆勾）
3. 填入對應時間
4. 確認「保持測試模式」是否勾選
5. 按下「儲存並啟動排程」

**啟動後，程式會先自動執行前置測試：**

- 自動開啟 Chrome，登入 NUEIP，嘗試找到打卡按鈕
- 找到後以紅框黃底標記（不點擊），彈出確認視窗
- **測試通過** → 關閉測試視窗後，正式排程啟動，進入倒數
- **測試失敗** → 顯示錯誤訊息，排程不啟動，可調整 XPath 後重試

6. 倒數期間狀態列顯示「[上班/下班] 倒數：N 秒」
7. 時間到時自動執行打卡

**時間已過的處理：** 若設定的打卡時間在當天已過，程式會自動將排程改為明天同一時間，並在日誌區顯示警告，不需要手動重設。

## 常見問題

**Q：測試模式彈窗後，關掉 Chrome 視窗，排程還會繼續嗎？**  
A：會。測試模式只是確認按鈕位置，關閉測試的 Chrome 視窗不影響排程倒數。

**Q：可以在沒有 GUI 的伺服器上跑嗎？**  
A：不行。程式使用 tkinter 圖形介面，需要桌面環境。若有無頭執行需求，需要另外改寫。

**Q：XPath 設定錯誤怎麼辦？**  
A：前置測試會在啟動排程前就報錯，可立即調整，不會等到打卡時間才發現問題。

**Q：`user_config.json` 裡的密碼是明文嗎？**  
A：是的，密碼以明文 JSON 儲存在本機。此檔案已加入 `.gitignore`，不會被上傳至版本庫。

**Q：打卡時間到，但 Chrome 遲遲沒有打開？**  
A：確認程式視窗沒有被意外關閉，並確認系統時間正確。ChromeDriver 首次使用時需要下載，若網路較慢可能有短暫延遲。
