# NUEIP 自動打卡神器

針對 [NUEIP 雲端人資系統](https://cloud.nueip.com) 設計的自動打卡工具。  
使用 Selenium 自動開啟 Chrome、填入帳密、定時點擊打卡按鈕，支援測試模式確認定位後再正式執行。

## 環境需求

| 項目 | 版本要求 |
|------|----------|
| Python | 3.8 以上 |
| Google Chrome | 最新穩定版 |
| ChromeDriver | 由 `webdriver-manager` 自動管理，無需手動下載 |

**必要套件：**

```
selenium
webdriver-manager
```

---

## 安裝步驟

1. 確認已安裝 Python 3.8+：

   ```bash
   python --version
   ```

2. 安裝相依套件：

   ```bash
   pip install selenium webdriver-manager
   ```

3. 執行程式：

   ```bash
   # v1p0
   python auto_check_out_v1p0.py

   # v1p1
   python auto_check_out_v1p1.py
   ```

## v1p0 使用說明

**檔案：** `auto_check_out_v1p0.py`  
**功能：** 下班自動打卡（單一排程）

### 介面說明

啟動後會出現圖形化視窗，分為以下三個區塊：

**第一步：登入資訊**

| 欄位 | 說明 |
|------|------|
| 公司代碼 | NUEIP 登入頁面的公司代碼 |
| 員工編號 | 你的員工帳號 |
| 登入密碼 | 你的登入密碼 |

> 帳號密碼會自動儲存於同目錄的 `user_config.json`，下次啟動自動讀取。

**第二步：設定時間與網址**

| 欄位 | 說明 | 預設值 |
|------|------|--------|
| 登入網址 | NUEIP 登入頁 URL | `https://cloud.nueip.com/login` |
| 設定時間 | 執行打卡的時間，格式 `HH:MM` | `18:00` |
| 測試模式 | 勾選後只標記按鈕位置，不實際點擊 | 預設開啟 |

> **注意：** v1p0 的實際觸發時間為設定時間 **+1 分鐘**。例如設定 `18:00`，實際會在 `18:01` 執行。

**進階 XPath 定位（選填）**

預設 XPath 已針對 NUEIP 頁面優化，一般不需要修改。如果 NUEIP 改版導致按鈕找不到，再調整這裡。

### 操作流程

1. 填入公司代碼、員工編號、密碼
2. 設定打卡時間
3. 確認「測試模式」是否勾選
4. 按下「儲存並啟動排程」
5. 程式進入倒數，狀態列會顯示剩餘秒數
6. 時間到時，Chrome 自動開啟、登入、執行打卡
   - **測試模式**：按鈕會標記紅框黃底，彈出確認視窗，不實際點擊
   - **正式模式**：直接點擊打卡按鈕，彈出完成通知

### 注意事項

- 若設定時間已過，程式不會自動改為隔天，必須重新設定
- 執行日誌僅輸出到 Python 終端機的 `print`，沒有記錄到檔案
- v1p0 僅支援**下班打卡**，沒有上班打卡功能

## v1p1 使用說明

**檔案：** `auto_check_out_v1p1.py`  
**功能：** 上下班雙排程自動打卡，含完整日誌與前置測試

### 介面說明

視窗比 v1p0 更大（680×960），新增了日誌顯示區與雙打卡設定。

**第一步：登入資訊（自動記憶）**

與 v1p0 相同。帳號密碼儲存於 `user_config.json`，但 v1p1 額外儲存打卡時間與啟用狀態。

**第二步：排程設定**

| 欄位 | 說明 | 預設值 |
|------|------|--------|
| 登入網址 | NUEIP 登入頁 URL | `https://cloud.nueip.com/login` |
| 上班打卡 | 勾選啟用；填入上班打卡時間 | 未勾選，`09:00` |
| 下班打卡 | 勾選啟用；填入下班打卡時間 | 已勾選，`18:00` |
| 保持測試模式 | 勾選後只標記按鈕，不實際點擊 | 預設開啟 |

> 上班與下班可同時勾選，程式會建立兩個獨立排程執行緒同時倒數。

**進階 XPath 定位（選填）**

比 v1p0 多一個「上班按鈕」XPath 欄位，其餘邏輯相同。

**執行日誌區**

視窗底部有 ScrolledText 日誌區塊，即時顯示程式執行狀態。所有日誌同步寫入同目錄的 `punch_log.txt`，可事後查閱。

### 操作流程

1. 填入公司代碼、員工編號、密碼
2. 勾選要排程的打卡項目（上班 / 下班 / 兩者皆勾）
3. 填入對應時間
4. 確認「保持測試模式」是否勾選
5. 按下「儲存並啟動排程」

**啟動後，程式會先自動執行前置測試：**

- 自動開啟 Chrome，登入 NUEIP，嘗試找到打卡按鈕
- 找到後以紅框黃底標記（不點擊），彈出確認視窗
- **測試通過** → 關閉測試視窗後，正式排程啟動，進入倒數
- **測試失敗** → 顯示錯誤訊息，排程不啟動，可調整 XPath 後重試

6. 倒數期間狀態列顯示「[上班/下班] 倒數：N 秒」
7. 時間到時自動執行打卡

### 時間已過的處理

如果設定的打卡時間在當天已過，v1p1 會自動將排程改為**明天同一時間**，並在日誌區顯示警告。不需要手動重設。

### 注意事項

- 至少需勾選上班或下班其中一項，否則無法啟動
- 日誌同步寫入 `punch_log.txt`，可事後查閱每次打卡紀錄
- 程式關閉後排程停止，請保持程式開啟直到打卡完成

## 版本更新說明（v1p0 → v1p1）

### 1. 新增上班打卡功能（雙排程）

v1p0 僅支援下班打卡（單一排程執行緒）。  
v1p1 新增上班打卡，並允許兩者同時啟動，各自以獨立執行緒倒數，互不干擾。

| | v1p0 | v1p1 |
|---|---|---|
| 下班打卡 | 支援 | 支援 |
| 上班打卡 | 不支援 | 新增 |
| 同時雙排程 | 不支援 | 支援 |

### 2. 完整執行日誌系統

v1p0 的狀態僅輸出到 Python 終端機的 `print`，沒有任何記錄。

v1p1 建立三層日誌機制：
- **Python `logging` 模組**：統一管理日誌等級（INFO / WARNING / ERROR）
- **`punch_log.txt`**：自動寫入磁碟，含時間戳記，可事後查閱
- **UI ScrolledText 區塊**：即時顯示在視窗底部，附 INFO / WARNING / ERROR 圖示

### 3. 啟動前自動執行測試（前置驗證）

v1p0 在啟動排程後直接進入倒數，等到時間到才執行，如果 XPath 設定錯誤，只有在打卡當下才會發現。

v1p1 在按下「儲存並啟動排程」後，立即自動執行一次測試模式：
- 確認帳密正確、頁面可登入
- 確認打卡按鈕可被定位
- **測試失敗則排程不啟動**，讓使用者提前修正設定，避免屆時打卡失敗

### 4. 時間已過自動改為隔天

v1p0 若設定時間早於目前時間，程式會立刻在 +1 分鐘後觸發（因為固定加1分鐘），行為不直觀。

v1p1 的 `_parse_trigger_time()` 明確偵測時間是否已過：
- 若未過 → 今天執行
- 若已過 → 自動改為明天同一時間，並在日誌記錄 WARNING

### 5. 倒數期間智慧 Sleep

v1p0 的等待迴圈固定每 0.5 秒輪詢一次，全程高頻輪詢。

v1p1 根據剩餘時間動態調整輪詢頻率：
- 距觸發 > 10 秒：大步跳（最多 sleep 5 秒），降低 CPU 空轉
- 距觸發 ≤ 10 秒：細粒度輪詢（0.5 秒），確保準時觸發

### 6. 設定檔儲存內容更完整

| 儲存欄位 | v1p0 | v1p1 |
|--------|------|------|
| 公司代碼 | 是 | 是 |
| 員工編號 | 是 | 是 |
| 密碼 | 是 | 是 |
| 登入網址 | 是 | 是 |
| 上班時間 | 否 | 是 |
| 下班時間 | 否 | 是 |
| 上班打卡啟用狀態 | 否 | 是 |
| 下班打卡啟用狀態 | 否 | 是 |

### 7. UI 與程式碼架構重構

v1p0 的 UI 建構邏輯直接寫在 `__init__` 中，混合了初始化與介面邏輯。

v1p1 將 UI 建構抽出為獨立的 `_build_ui()` 方法，並引入執行緒鎖（`_tasks_lock`）管理多工任務計數，在所有排程完成後正確重設 UI 狀態。

## 常見問題

**Q：測試模式彈窗後，關掉 Chrome 視窗，排程還會繼續嗎？**  
A：會。測試模式只是確認按鈕位置，關閉測試的 Chrome 視窗不影響排程倒數。

**Q：可以在沒有 GUI 的伺服器上跑嗎？**  
A：不行。程式使用 tkinter 圖形介面，需要桌面環境。若有無頭執行需求，需要另外改寫。

**Q：XPath 設定錯誤怎麼辦？**  
A：v1p1 的前置測試會在啟動排程前就報錯，可立即調整。v1p0 則要等到排程觸發時才會顯示錯誤。

**Q：`user_config.json` 裡的密碼是明文嗎？**  
A：是的，密碼以明文 JSON 儲存在本機，請確保該檔案不要上傳至公開儲存庫或分享給他人。

**Q：打卡時間到，但 Chrome 遲遲沒有打開？**  
A：確認程式視窗沒有被意外關閉，並確認系統時間正確。ChromeDriver 首次使用時需要下載，若網路較慢可能有短暫延遲。
