# NUEIP 自動打卡神器

針對 [NUEIP 雲端人資](https://cloud.nueip.com) 的自動登入與打卡工具，支援上班 / 下班雙排程，排程時間到自動開啟 Chrome 完成打卡。

> 目前最新版本：**v1p2** (`auto_check_out_v1p2.py`)

---

## 系統需求

| 項目 | 需求 |
|---|---|
| Python | 3.9 以上 |
| Google Chrome | 已安裝（版本不限，會自動匹配 ChromeDriver） |
| 作業系統 | Windows / macOS / Linux |

### 安裝相依套件

```bash
pip install selenium webdriver-manager
```

---

## 快速開始

```bash
python auto_check_out_v1p2.py
```

---

## 介面說明

### 第一步：登入資訊

| 欄位 | 說明 |
|---|---|
| 公司代碼 | NUEIP 登入頁的公司代碼欄位 |
| 員工編號 | 登入用帳號 |
| 登入密碼 | 登入用密碼（輸入後以 `*` 遮蔽） |

輸入後點擊「儲存並啟動排程」會自動存入 `user_config.json`，下次開啟程式自動帶入。

### 第二步：排程設定

| 欄位 | 說明 |
|---|---|
| 登入網址 | 預設 `https://cloud.nueip.com/login`，通常不需修改 |
| 上班打卡 | 勾選後啟用，設定時間格式 `HH:MM`（例如 `09:00`） |
| 下班打卡 | 勾選後啟用，設定時間格式 `HH:MM`（例如 `18:00`） |
| 保持測試模式 | 勾選 = 只標記按鈕、不實際點擊（建議首次使用時保持開啟） |

> **時間已過自動順延**：若設定時間早於當前時間，程式會自動改為隔天同一時間執行，並在日誌中顯示 WARNING。

### 進階 XPath 定位

預設值已針對 NUEIP 現行版本優化，一般不需修改。若 NUEIP 改版導致定位失敗，可在此欄位填入從 Chrome DevTools（F12）複製的 XPath。

## 執行流程

```
點擊「儲存並啟動排程」
        ↓
自動執行測試模式（確認能找到打卡按鈕）
        ↓
測試通過 → 啟動背景倒數排程
        ↓
排程時間到 → 開啟 Chrome → 自動登入 → 點擊打卡按鈕
        ↓
打卡完成，Chrome 視窗保持開啟供確認
```

### 測試模式 vs 正式模式

| 模式 | 行為 |
|---|---|
| 測試模式（預設） | 登入後找到打卡按鈕，以**紅框黃底**標記，不點擊，彈窗提示確認位置 |
| 正式模式 | 實際點擊打卡按鈕完成打卡 |

首次使用建議保持測試模式，確認紅框標記在正確的按鈕上，再取消勾選切換為正式模式。

---

## XPath Fallback 機制（v1p2 新增）

當預設 XPath 無法定位打卡按鈕時，程式會依序嘗試 4 組候選 XPath，並在日誌中顯示哪一組成功命中：

```
[上班] XPath 候選 #1 定位成功   ← 正常情況
[上班] XPath 候選 #1 未命中，嘗試下一組…
[上班] XPath 候選 #2 未命中，嘗試下一組…
[上班] XPath 候選 #3 定位成功   ← fallback 命中
```

若 4 組全部失敗，錯誤訊息會提示需要手動更新 XPath。

---

## 執行日誌

- **UI 即時顯示**：視窗下方的日誌區塊，前綴 `i` = INFO，`!` = WARNING，`X` = ERROR
- **檔案記錄**：同步寫入 `punch_log.txt`，格式為 `時間戳記 [等級] 訊息`

---

## 注意事項

- **密碼以明文儲存**於 `user_config.json`，請勿在公用電腦使用或將此檔案分享給他人
- 程式執行期間請保持電腦不休眠，否則 Chrome 可能無法在排程時間正常啟動
- Chrome 版本升級後，`webdriver-manager` 會自動下載對應的 ChromeDriver，不需手動更新
